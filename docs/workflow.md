# BlueSentinel - User Workflows

This document outlines the primary user journeys within the BlueSentinel platform.

---

## 1. Monitoring Workflow (Dashboard)

**Goal**: Analyze real-time water quality data and identify anomalies.

1. **Entry**: User navigates to `dashboard.html` from the Landing Page or via Direct Link.
2. **Overview**:
    - User sees the **Bento Grid** with live sensor cards (Temperature, pH, Turbidity, DO).
    - Status pills indicate "Normal", "Warning", or "Critical" states immediately.
3. **Deep Dive**:
    - User hovers over a graph to see historical trends (last 30 points).
    - User checks the **AI Projection Chart** to see the predicted "Quality Index".
4. **Action**:
    - If a card shows "Critical", a **"View Solution"** button appears.
    - Clicking it opens the **AI Assistant** (SentinelBuddy) with specific remediation advice (e.g., "Add buffering agents").
5. **Analysis**:
    - User clicks **"Refresh Analysis"** on the Eco-System card to get a text summary generated by Gemini AI.

---

## 2. Community Reporting Workflow (Forums)

**Goal**: Submit manual field observations to the community.

1. **Entry**: User clicks the "Forums" icon in the **Floating Dock**.
2. **Browse**: User sees pinned announcements and recent field reports from other researchers.
3. **Submit**:
    - User clicks **"+ New Field Report"**.
    - A form expands.
    - User enters localized data (Temperature, pH, Notes).
    - User clicks **"Submit to Community"**.
4. **Feedback**:
    - The new post appears instantly in the thread list (Real-time Firebase update).
    - A success notification is shown.

---

## 3. Incident Investigation Workflow (Logs)

**Goal**: Review historical alerts and geographic incidents.

1. **Entry**: User clicks the "Logs" icon in the **Floating Dock**.
2. **Visualization**:
    - User interacts with the **3D Globe**.
    - User spots a glowing red node (Critical Alert).
3. **Investigation**:
    - User clicks the node to focus the camera on that region.
    - User reviews the detailed **Log Table** below to match the timestamp and error type (e.g., "Turbidity Spike at Delta-Z").

---

## 4. Information Gathering Workflow (News)

**Goal**: Stay updated on global river conservation efforts.

1. **Entry**: User clicks the "News" icon.
2. **Consumption**:
    - User scrolls through the masonry grid of news articles.
    - Articles are fetched dynamically (simulated or API).
    - User clicks "Read More" to visit the source.

---

## 5. Knowledge & Contact Workflow

**Goal**: Understand the system architecture or contact the team.

1. **Entry**: User navigates to `contact.html` (or clicks footer links).
2. **Learning**:
    - User scrolls through the **Workflow**, **Architecture**, and **Tech Stack** sections.
    - Animations reveal content progressively.
3. **Contact**:
    - User fills out the contact form.
    - Message is sent to Firebase `/messages`.
    - User receives an immediate "Sent!" confirmation.
